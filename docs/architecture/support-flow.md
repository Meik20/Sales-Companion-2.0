# Support Flow

## User side

- création d'un thread de support
- type `chat` pour échange rapide
- type `ticket` pour demande nécessitant réflexion
- messages stockés en sous-collection `messages`

## Admin side

- visualise tous les threads
- répond dans la messagerie
- peut traiter les tickets ouverts
- fonctionne en temps réel via Firestore
